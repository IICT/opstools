Share folders with your VM using only the command line

by IICT Team

Posted on Wed, Feb 24, 2016

Installing VirtualBox Guest Additions (on a server version)

You need to install the VirtualBox Guest Additions to use different useful tools, like shared folders, mouse integration or the full-screen support. On a server edition though, the only thing we want is the ability to work with shared folders.

Installing the tools

  1. Start by Going in the VirtualBox menu on your VM, go to Devices -> Insert Guest Additions CD image…

  2. Install the following dependencies :

    $ sudo apt-get update && sudo apt-get install dkms build-essential linux-headers-$(uname -r)

  3. Mount the CD:

    $ sudo mount /dev/cdrom /media/cdrom
# You may have a read-only warning, it's normal, CD are by definition read-only devices.

    You can test it like so:

    $ ls -l /media/cdrom/

    You should see the content of the CD.

  4. Install the Guest Additions

    $ sudo /media/cdrom/VBoxLinuxAdditions.run

    You may have this warning:

    Installing the Window System drivers ...fail!
(Could not find the X.Org or XFree86 Window System.)

    You can just ignore it.

Shared folder

Having a shared folder is a good way to share stuff between your host and your guest VM.

  1. On your VirtualBox menu, go to Devices -> Shared Folder -> Shared Folder Settings…

Select the folder you want to use a your share, and select auto-mount.

  1. On the VM, type in this command:

    $ sudo adduser <username> vboxsf

  2. Reboot the VM

    $ sudo reboot

  3. The mount should be done, you can see where it is like this:

    $ mount | grep vbox

  4. You should be able to see the content

    $ ls -l /media/sf_VBoxShared (name should be sf_ShareName)

And you are done.
