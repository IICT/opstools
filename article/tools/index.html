VirtualBox Tools

by IICT Team

Posted on Fri, Feb 19, 2016

Customize your installation

Install the VirtualBox Tools

By default, a virtualized OS doest not know that it is virtual. It strongly believes that it is installed directly on top of hardware and that it is in total control. Well, that’s not true. The VirtualBox tools are meant to be installed on a VM, to enable some options that are not possible otherwise.

In the menu of VirtualBox -> Devices -> Insert Guest Additions CD Image…

  • Select Run
  • Type in your password

  • reboot

    $ sudo reboot

You now have support for the fullscreen on your VM. (Host + f).

Enable sharing of folders between the Linux VM and your host machine

On the command line of the Linux VM run the following commands and answer all questions with the default values:

$ sudo apt-get update
$ sudo apt-get upgrade
$ sudo apt-get install dkms

Reboot the VM:

$ sudo shutdown -r now

Insert into the VM’s virtual CD-ROM drive the CD file VBoxGuestAdditions.iso: Navigate to Devices > CD/DVD Devices > Choose a virtual CD/DVD disk file…

  • On Windows:
  • On Mac: Leave the file selection dialog open. Open another finder window and navigate to Applications > VirtualBox.app. Bring up the context menu ande select Show package contents. Navigate to Contents > MacOS > VBoxGuestAdditions.iso. Drag-and-drop this file into the open file selection dialog.

Install the guest additions:

$ sudo mount -t iso9660 /dev/cdrom /media/cdrom
$ sudo bash /media/cdrom/VBoxLinuxAdditions.run
$ sudo umount /media/cdrom

Share a folder on the host machine: Navigate to Devices > Shared Folder Settings… Click on Add. Select the folder you want to share, for example your home directory. Give a name under which this shared folder should appear to the guest, say albert.einstein.

  • Enable Read-only if you want to prevent accidental modification of your files from Linux.
  • Enable Auto-mount

Click OK.

On the Linux VM create a directory where you want to mount the shared folder:

$ mkdir ~/hostfiles

Mount the shared folder albert.einstein on the directory hostfiles:

$ sudo mount -t vboxsf -o uid=$UID,gid=$(id -g) albert.einstein ~/hostfiles

Install an other GUI for your VM

Usually, you wouldn’t set a GUI (Graphical User Interface) on a server. Still, the idea here is to make you discover more about Linux and how you can use it. The problem with the default GUI you get with Ubuntu is the fact that it’s quite heavy and laggy as a VM. You can pick a different one.

You will find a list of the diffrent GUI for Ubuntu and how to install them. We recommend that you use Xfce or LXDE, both are pretty light and simple to use.

You can also have various GUI installed at the same time and pick the one you want, in the login page, there is a white circle next to your name, clicking on it will allow you to choose the one you want.

Installing Xfce

Install only the desktop environment

$ sudo apt-get install xubuntu-desktop
$ sudo reboot

You may need to change the keyboard configuration

Installing LXDE

$ sudo apt-get install lxde 
$ sudo reboot
