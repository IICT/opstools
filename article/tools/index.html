Integrate your virtual machine with your local machine

by IICT Team

Posted on Fri, Feb 19, 2016

VirtualBox customisations

Install the VirtualBox Guest Additions

By default, a virtualised OS doest not know that it is running in a virtual machine. It believes that it is installed directly on top of hardware and that it is in total control of the keyboard and mouse, for example. Well, that’s not true. The VirtualBox tools are meant to be installed on a VM, to enable some options that are not possible otherwise, like correct handling of the mouse, sharing of files between guest and host and others.

In the menu of VirtualBox -> Devices -> Insert Guest Additions CD Image…

  • Select Run
  • Type in your password

  • reboot

    $ sudo reboot

You now can run the VM in a window with proper handling of the mouse or you can run it in fullscreen mode (Host + f).

Enable bidirectional clipboard (copy & paste between host and guest)

This is a very useful tool that will allow you to share your clipboard between your host and your guest machines. To do this, go to Devices, Shared Clipboard, Bidirectional.

You probably need to install the VirtualBox Guest Additions, as explained in the previous chapter.

Pro tips: usually, the ctrl+c and ctrl+v do not work well between the host and the guest, use right click copy and right click paste.

Enable sharing of folders between the Linux VM and your host machine

Normally, as the VM runs its own operating system, your host machines sees the VM as a completely separate machine. To transfer files between host and guest you would need to transfer the files over the network, for example using a file transfer protocol that is difficult to set up. Luckily there is a simpler solution: VirtualBox offers an option to share a folder between host and guest. This file sharing works similar to network file sharing, but is much easier to set up.

On the command line of the Linux VM run the following commands and answer all questions with the default values:

$ sudo apt-get update
$ sudo apt-get upgrade
$ sudo apt-get install dkms

Reboot the VM:

$ sudo shutdown -r now (or sudo reboot)

Navigate to Devices > Insert Guest Additions CD image… This will mount a CD on your Desktop.

You can run find the CD on the file explorer (name may change depending on your Desktop version). You should have something like:

  • Open Autorun Prompt
  • Run Program / script

If not, double click on the VBoxLinuxAdditions.sh.

This will open a terminal and install some of the needed packages. You can restart your machine after that.

$ sudo reboot

Share a folder on the host machine: Navigate to Devices > Shared Folder Settings… Click on Add. Select the folder you want to share, let’s say you just created a folder called C:\Users\Your_Name\ADS (you can put it wherever you want, like My Documents or your Desktop).

  • Enable Read-only if you want to prevent accidental modification of your files from Linux. (Not recommended if you want to share files from your VM to your host machine).
  • Enable Auto-mount

Click OK.

Now, on your VM, type in this command (it will add you on the group that can work with the shared folder):

# Use your username, administrator by default if you followed the guide.
$ sudo adduser <username> vboxsf

Now you only need to restart your machine, from now on, Ubuntu will auto-mount your shared folder.

Install an other GUI for your Ubuntu Linux

The GUI you get with Ubuntu (called Unity) tends to run slowly inside a VM. Fortunately in the Linux world you can choose between many different GUIs. Some of them are faster, and some of them try to emulate the look and feel of Windows.

The creators of Ubuntu maintain a list of the different GUI for Ubuntu and how to install them. We recommend Xfce or LXDE, both are pretty light and simple to use.

You can also have different GUIs installed at the same time and pick the one you want when you log in. In the login page there is a white circle next to your name, clicking on it will allow you to choose the one you want. If this circle is not there, sometimes the icon is somewhere else, like on the right top of the screen.

Installing Xfce

Install only the desktop environment

$ sudo apt-get install xubuntu-desktop
$ sudo reboot

You may need to change the keyboard configuration

Installing LXDE

$ sudo apt-get install lxde 
$ sudo reboot
