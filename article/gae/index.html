Preparing a Google App Engine development environment

Google App Engine offers a free usage tier that gives us ample room to perform interesting labs.

To set up a development environment for Google App Engine, you need to perform two things:

  1. Set up a Google account for the use of Google App Engine.

  2. Download and install the Eclipse-based Google App Engine development environment.

These two tasks are detailed below.

Set up a Google account

You can either use an existing Google account (if you are using one of Google’s services such as GMail you already have an account) or you may decide to create a separate account. We recommend to use separate accounts for private and study-related activities.

  • In your browser navigate to http://appengine.google.com.

  • If you want to use an existing account, use the Sign in form, otherwise click on Create account and follow the instructions.

  • You will be asked to provide a mobile phone number for verification purposes. Follow the instructions to receive a code on your phone and enter the code on the web page.

When the setup is completed you should see the page Welcome to Google App Engine.

Set up the development environment on your machine

The setup of the development environment consists of the steps detailed below. If you run into trouble, please consult the Google App Engine Java Getting Started guide.

  1. Make sure your computer has Java SE Development Kit (JDK) 7 installed. As of March 2016 Google App Engine does not accept bytecode produced for Java 8. You can install JDK 7 and 8 side-by-side on your computer, see the section Running Java 7 and Java 8 side-by-side below. The last public release of JDK 7 from Oracle is Java SE Development Kit 7u80.

  2. Download and install Eclipse IDE for Java EE Developers version 4.4.x/Luna that you will find on the Eclipse Luna SR2 packages download page.

  3. Download and install the Google Plug-In for Eclipse and the Google App Engine SDK by using the plug-in installer built into Eclispe:

    • Start Eclipse

    • Select Help > Install New Software

    • In the Work with: field enter https://dl.google.com/eclipse/plugin/4.4 and type Enter.

      • If you are using version 4.3 of Eclipse, use the URL https://dl.google.com/eclipse/plugin/4.3, if you are using version 3.8 or 4.2, use https://dl.google.com/eclipse/plugin/4.2.

    • Check Google Plugin for Eclipse

    • Check SDKs > Google App Engine Java SDK

    • Click Next and follow the instructions. When the security warning appears “You are installing software that contains unsigned content” click OK.

  4. Verify the proper installation of Eclipse and the plug-in:

    • Verify Eclipse version: Select Eclipse > About Eclipse

    • Verify that the GAE plug-in is active: Select File > New Project… > Google > Web Application Project. Click Next. You should see the dialog Create a Web Application Project. Click Cancel.

Running Java 7 and Java 8 side-by-side

If you already have Java 8 on your local machine and you installed Java 7 you can start Eclipse using the one or the other. Typically Eclipse (and other Java applications) look for the JAVA_HOME variable to find the Java runtime.

Setting the Java version on Windows

Write the following batch file and start Eclipse by running it:

SET JAVA_HOME="C:\Program Files\Java\jdk1.7.0_80"
SET PATH="%JAVA_HOME%\bin"
START eclipse.exe

or edit the file eclipse.ini to contain the line:

-vm C:\Program Files\Java\jdk1.7.0_80\bin\java.exe

Setting the Java version on Mac OS X

Add the following line to the .bash_profile file in your personal directory:

export JAVA_HOME=/Library/Java/JavaVirtualMachines/<jdk_version>/Contents/Home

where <jdk_version> is your JDK version, for example, jdk1.7.0_80.jdk.

To make the change immediately effective in your current bash session, run

source ~/.bash_profile

Then start Eclipse from the command line.

Setting the Java version on Linux

Add the following line to the .profile file in your personal directory:

export JAVA_HOME=/usr/local/tools/java/<jdk_version>

where <jdk_version> is your JDK version, for example, jdk1.7.0_80.jdk.

To make the change immediately effective in your current bash session, run

source ~/.profile

Then start Eclipse from the command line.
