Quick guide to Markdown

by IICT Team

Posted on Wed, Feb 24, 2016

What is Markdown

Markdown is a very lightweight markup language (like HTML) that is being used by a lot of people. It’s fast, efficient, really easy to read and compatible pretty much everywhere.

There are many websites and examples where the text is written in Markdown:

  • READMEs and documentation files on github are in Markdown (extension .md)
  • Some of the documentation on Moodle is written in Markdown.
  • This very article is also written in Markdown (and obviously all the other articles on this blog).

Quick guide

Here is a quick guide for the syntax of Markdown

Titles

To make a title, you need to use the # character. It works like HTML title, h1 is the biggest, then comes h2, then h3… In Markdown, it’s the number of # you put that defines the level:

# h1
## h2
### h3 
#### h4 
...

So, for example:

# My big title
## My not so big title
### My small title
...

will give you:

My big title

My not so big title

My small title

Etc…

Note: be sure to put a <space> between the last # and your first letter. If you don’t, it may give you weird results on some browsers.

Text modification

These modifications would be this or this. Once again, to do this in Markdown, it’s really easy.

Bold

**Your text**

or

__Your text__

Both of these will give the same result.

Italic

*Your text*

or

_Your text_

Quite easy.

Note: Usually, we want to separate the special characters and the text with a <space>, like with titles, here we do the opposite. We want the text to be right after the character!

Lists

  • This
  • Or this
    • Or that
    • And that too
      • Maybe this too?

How to do this? Once again, it’s simple, it’s the * character.

* This
* Or this 
 * Or that
 * And that too
   * Maybe this too?

To create a list, put a star, followed by a <space> and your text. To create a new element on the same of the list, just do the same and finally, to indent it, use a tab or 2 <spaces> characters before putting the first star.

* 1
* 2
  * 2.5 <-- Tab or two spaces before the star

Code

This type of “box”:

This one here

is used to type in “code”. We use if for commands, for instance:

$ sudo reboot

To do so, just start your line with a <tab> or 4 <spaces>. (Always use tab)

Not easy to explain 
    new tab
        other tab

You can also indent this with other <tab> characters.

Quote

This is a quote:

Hey, I am a quote

To create this, just start your line with >.

> a quote
> a second quote

Images and links

You can add images and links in your documents with these commands:

Link

[Descrition](http://your_url.ch)

Image

![Description](http://your_image.png)

Obviously, you can create absolute or relatives links, meaning you can link something by it’s url or in your working directory, like this:

# Your image is in the "img" folder
![My super image](./img/super_image.png)

HTML

Markdown is made to work with HTML, you can type in HTML wherever you want. For instance, there is no support for tables in Markdown, so if you want to create one, you can use html tags.

<table>
    <tr>
        <th>header1</th>
    </tr>
    <tr>
        <td>Content1</td>
    </tr>
<table>

So be careful, every time you want to write text between < and > (Like an html tag), it will be interpreted as a HTML tag and may give you strange behaviours… Unless you escape it!

Escaping characters

You should have noticed, Markdown is really easy, we work a lot with tabulations, *, and _. But what happens when you write something like:

your_user_name

it will give you this: your_user_name

The _ characters will be used to render the “user” text in italic, because they have not been escaped. We need to give the indication that we want the _ to be interpreted as text. How to do this? We use the escape character, .

your\_user_name

will give you: your_user_name.

We could obviously also escape the second one:

your\_user\_name

but it’s not necessary, because a simple _ will be interpreted as text anyway.

The same happens when you want to use a star, let’s say:

rm -r *march.*

rm -r march.

The stars were not treated as text, so escaping the first star should help, like we did previously.

rm -r \*march.* OR rm -r \*march.\*

This gives you: rm -r *march.*

There are some other cases where you need to escape the escape character itself to print it, but that should be enough information for now :)

And also, the same applies to the < and > characters, if you want to print this:

<my_tag>

You have to write it like this:

\<my_tag>

So you can see it (otherwise it will considered like a HTML tag and it won’t be printed. Even worse, depending on the tag, it may even give you problems, because it will be expecting the closing tag, like </my_tag>.
